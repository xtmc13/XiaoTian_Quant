## 4. 关键结论

### 4.1 修复有效性

**已验证通过**: 15 个模块的字段映射和返回格式修复已生效，新 Gateway (8081) 返回的数据格式与前端 TypeScript 类型定义匹配。

### 4.2 路径不匹配是最大问题

前端和后端存在 **6 处 API 路径不匹配**，导致以下页面无法正常工作：
- 指标 IDE 页面 (`/api/indicators` vs `/api/indicator/list`)
- 策略管理页面 (`/api/strategies` vs `/api/strategies/configs`)
- 持仓页面 (`/api/positions` vs `/api/portfolio/positions`)
- Agent 设置页面 (`/api/settings/agent-models` vs `/api/settings/agent/models`)
- 实验页面 (`/api/experiments` 无对应路由)
- 链上数据页面 (`/api/onchain/...` 无对应路由)

### 4.3 建议修复方案

**方案 A: 后端添加别名路由**（推荐）
在 `cmd/server/main.go` 中为前端调用的路径添加别名：
```go
// 指标别名
api.GET("/indicators", indicator.ListIndicators)

// 策略别名
api.GET("/strategies", handler.GetStrategyConfigs)

// 持仓别名
api.GET("/positions", handler.PortfolioPositions)

// 设置别名
api.GET("/settings/agent-models", handler.SettingsAgentModels)
```

**方案 B: 前端修改 API 路径**
修改前端 `api.ts` 中的路径调用，匹配后端实际路由。

**建议**: 采用方案 A，因为前端路径更直观（RESTful 风格），且改动量更小。

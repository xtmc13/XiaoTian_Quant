## 一、高优先级

### 1.1 WebSocket 断线重连增强
**问题：** 币安 WS 断开后，K 线出现断层，需手动刷新页面
**方案：** 
- 在 datafeed 中记录最后一条 K 线时间戳
- 重连后自动拉取缺失的 K 线数据（REST API 补数据）
- 图表无感恢复，不需要用户操作

### 1.2 回测引擎接入真实数据
**问题：** 当前回测用随机生成的模拟数据，结果无参考价值
**方案：**
- 回测时自动从币安拉取历史 K 线
- 支持选择日期范围（1 个月 / 3 个月 / 6 个月 / 1 年）
- 向量化回测用 `run_vectorized_backtest`，事件驱动用 `BacktestExchange`

### 1.3 策略社区（发布 / 评论 / 过拟合检测）
**来源：** QuantDinger `indicator-community`
**方案：**
- 新增"策略社区"页面
- 用户可发布策略（名称 + 代码 + 回测结果）
- 其他用户可评论、点赞
- 自动计算过拟合风险指标（样本内 vs 样本外对比）

## 二、中优先级

### 2.1 Monaco 代码编辑器
**问题：** 当前用 `<textarea>`，无语法高亮、无代码补全
**方案：**
- CDN 引入 Monaco Editor
- Agent 策略生成页面的代码区替换为 Monaco
- 支持 Python 语法高亮 + 自动补全 + 错误提示

### 2.2 券商账户连接
**来源：** QuantDinger `broker-accounts`
**方案：**
- 设置页新增"券商账户"标签
- 支持连接：币安、OKX、Alpaca、IBKR、MT5
- API Key 加密存储

### 2.3 资产管理页面
**问题：** 当前只有交易页的持仓卡片，缺少独立的资产管理视图
**方案：**
- 新增"资产"页面（或作为交易页的子标签）
- 持仓列表 + 盈亏明细 + 历史收益曲线
- 多账户聚合视图

### 2.4 移动端适配
**问题：** 当前界面仅适配桌面端，手机上完全不可用
**方案：**
- 添加响应式 CSS（`@media` 断点）
- 手机端：单列布局 + 底部导航 + 简化图表
- 平板端：双列布局

## 三、低优先级

### 3.1 前后端分离
**问题：** `web/app.py` 8000+ 行，HTML/CSS/JS 全嵌在 Python 字符串里
**方案：**
- 将前端 SPA 拆出为独立的 `index.html` + `static/` 目录
- FastAPI 用 `FileResponse` 提供静态文件
- 路由拆分到 `routes/` 下的独立模块

### 3.2 策略参数优化（Hyperopt）
**来源：** FreqTrade Hyperopt
**方案：**
- 对向量化策略自动搜索最优参数
- 支持：RSI 周期、EMA 周期、止损比例、仓位大小
- 用网格搜索或贝叶斯优化

### 3.3 多语言完整覆盖
**问题：** 当前双语只覆盖了主要 UI 元素
**方案：**
- `策略` 页面、`设置` 页面、`回测` 页面补充 data-i18n
- Toast 提示、错误消息也走翻译

### 3.4 通知中心
**问题：** 告警只能通过弹窗，无历史记录
**方案：**
- 顶部导航栏新增通知铃铛图标
- 下拉列表显示最近的告警（回撤告警、成交提醒、策略信号）
- 支持标记已读 / 全部清除

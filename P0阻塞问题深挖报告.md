### 2.1 文件位置

```
  - 第 282-291 行：handleStockSearch 函数（本地数组过滤）
```

### 2.2 硬编码数据逐项拆解

#### A. 市场情绪指标（第 34 行）


**问题**：这三个指标从未被任何 API 调用更新。用户看到的永远是相同的数字。

**问题**：8 个全球主要指数价格全部写死。`loadMarketData` 函数完全没有尝试获取这些指数的真实价格。

#### C. 热力图（第 45-93 行）—— 6 大类 72 个标的

|------|--------|------|
| `us_stocks` | 12 | 价格/涨跌幅全部写死 |
| `hk_stocks` | 12 | 价格/涨跌幅全部写死 |

**关键代码**（第 161-190 行）：


**问题**：`loadMarketData` 只请求了 6 个 crypto 的 snapshot，其他 66 个标的（美股、港股、大宗商品、板块、外汇）永远显示写死数据。

#### D. 经济日历（第 95-101 行）


### 2.3 修复方案

**方案 A：最小修复（推荐立即执行）**

1. **移除所有初始硬编码数据**，改为空数组/null：
   ```tsx
   const [marketData, setMarketData] = useState({
     fearGreed: undefined, vix: undefined, dxy: undefined,
     indices: [], heatmap: { us_stocks:[], hk_stocks:[], crypto:[], commodities:[], sectors:[], forex:[] },
     calendar: [],
   })
   ```

2. **扩展 `loadMarketData`**：
   - 调用 `marketApi.indices()` 获取全球指数
   - 调用 `marketApi.heatmap(type)` 获取各类热力图
   - 调用 `marketApi.calendar()` 获取经济日历
   - 调用 `marketApi.sentiment()` 获取恐惧贪婪/VIX/DXY

3. **添加骨架屏/加载态**：数据加载中显示 `Skeleton` 而非虚假数字。

**方案 B：完整修复**
- 后端新增 `/api/market/indices`、`/api/market/heatmap`、`/api/market/calendar`、`/api/market/sentiment` 接口
- 前端对接上述接口
- 热力图按类别分批加载，减少单次请求压力

**预估工时**：方案 A 1.5 天（纯前端），方案 B 3 天（含后端接口）

## 四、问题 3：行情 fallback 硬编码误导数据

### 4.1 文件位置

```

  - 第 53-58 行：AISnapshot 默认值
  - 第 60-87 行：fetchBinance24hrTicker 调用与覆盖逻辑
```

### 4.2 代码分析

```go
func AISnapshot(c *gin.Context) {
  symbol := c.DefaultQuery("symbol", "BTCUSDT")
  interval := c.DefaultQuery("interval", "1h")

  // 默认值 —— 无论请求什么标的，失败时都返回 BTC 的价格
  priceRaw := 68000.0

    // 只有成功时才覆盖默认值
```

### 4.3 风险场景

| 场景 | 后果 |
|------|------|
| Binance API 限流/超时 | 用户查询 ETH，返回 BTC 的价格（68000），严重误导 |

```go
func AISnapshot(c *gin.Context) {
  symbol := c.DefaultQuery("symbol", "BTCUSDT")
  
  ticker, err := fetchBinance24hrTicker(symbol)
  if err != nil {
    c.JSON(http.StatusServiceUnavailable, gin.H{
      "status": "error",
      "msg": "Market data temporarily unavailable: " + err.Error(),
    })
    return
  }
  
  // 解析 ticker...
}
```

**完整修复**：
- 支持多交易所 fallback（Binance → OKX → Coinbase）
- 添加本地缓存，减少 API 依赖
- 对非 crypto 标的调用对应数据源（股票用 AlphaVantage/Yahoo，外汇用特定源）

**预估工时**：最小修复 0.1 天，完整修复 1 天

## 五、问题 4：AI 页面股票搜索本地硬编码

### 5.1 文件位置

```
web/src/pages/AI/index.tsx
  - 第 282-291 行：handleStockSearch
web/src/pages/AI/components/Modals.tsx
  - 第 64 行：热门标的硬编码
```

### 5.2 代码分析

```tsx
const handleStockSearch = useCallback(() => {
  const q = stockSearchQuery.trim().toUpperCase()
  if (!q) return
  const all = ['AAPL', 'MSFT', 'NVDA', 'TSLA', 'GOOGL', 'AMZN', 'META', 'BTC/USDT', 'ETH/USDT', 'BNB/USDT', 'SOL/USDT', 'XRP/USDT']
  const matched = all.filter((s) => s.includes(q))
  if (matched.length === 0 && !q.includes('/')) {
    matched.push(q + '/USDT')
  }
  setStockSearchResults(matched)
}, [stockSearchQuery])
```

**问题**：
1. 搜索池只有 12 个硬编码标的
2. 没有调用任何搜索 API（如 `/api/market/search?q=...`）
3. 如果用户输入不存在于 `all` 中的代码（如 `BABA`、`TCEHY`），只会机械地追加 `/USDT`
4. 热门标的也是写死的 6 个

### 5.3 修复方案

**最小修复**：

```tsx
const handleStockSearch = useCallback(async () => {
  const q = stockSearchQuery.trim().toUpperCase()
  if (!q) return
  try {
    const res = await marketApi.search(q) // 调用后端搜索接口
    setStockSearchResults(res.symbols || [])
  } catch {
    // fallback：本地模糊匹配（扩大标的池到 Top 100）
    setStockSearchResults(localSearch(q))
  }
}, [stockSearchQuery])
```

**后端需新增**：`GET /api/market/search?q={query}` 接口，对接交易所 symbol 列表或本地数据库。

**预估工时**：前端 0.5 天 + 后端接口 0.5 天 = 1 天

## 二、重复代码（DRY 违规）

### 🔴 严重重复 — 多处大面积复制粘贴

#### 1. CRA 量化策略参数表单（~400 行 × 5 处）

以下文件包含 **几乎完全相同的 CRA 参数配置 UI**：
- `Settings.tsx`（策略设置 Tab）
- `Strategy.tsx`
- `Bots.tsx`
- `Backtest.tsx`
- `IndicatorIDE.tsx`（参数面板）

重复字段包括：订单数量、首单金额、加仓间隔、回调比例、止盈比例、利润回调、止盈方式、开仓指标、加仓指标、瀑布式、开仓加倍、趋势指标等 **15+ 个字段**。

**建议**：提取为 `<CRAParamForm config={...} onChange={...} />` 共享组件。

#### 2. 现货 / 合约交易页面（~600 行 × 2 处）

`SpotTrading.tsx` 与 `ContractTrading.tsx` 存在大量重复：

| 重复内容 | 行数估算 |
|----------|---------|
| `formatPrice / formatTime / formatDateTime / formatVolume` | ~50 行 |
| `StatusTag` 组件 | ~15 行 |
| `obFormatPrice` 函数 | ~10 行 |
| `KLineChartPro` 初始化 + 周期监听 + 清理逻辑 | ~80 行 |
| Orderbook 渲染（asks/bids + 深度图 + spread） | ~120 行 |
| WebSocket 价格/成交监听 | ~30 行 |
| 底部面板 Tab 切换 + 拖拽调整高度 | ~60 行 |
| 订单列表 / 历史 / 成交记录表格 | ~150 行 |

**建议**：提取为共享 hooks (`useTradingChart`, `useOrderBook`) 和组件 (`<OrderBookPanel>`, `<TradeHistoryPanel>`)。

#### 3. ECharts 懒加载辅助函数

以下文件各自定义了相同的 `getEcharts()`：
- `Dashboard.tsx`
- `Portfolio.tsx`
- `Backtest.tsx`

```ts
let echartsLib: any = null
async function getEcharts() {
  if (!echartsLib) echartsLib = await import('echarts')
  return echartsLib
}
```

**建议**：移至 `src/lib/echarts.ts` 统一导出。

#### 4. 时间周期常量

`SpotTrading.tsx`、`ContractTrading.tsx`、`ModelManagement.tsx`、`HyperoptManagement.tsx` 各自定义了 `INTERVALS` 数组。

**建议**：在 `src/lib/constants.ts` 中统一定义。

## 三、不完善 / 缺失的功能

### 🔴 功能缺失

| 功能 | 现状 | 影响 |
|------|------|------|
| **指标发布流程** | `IndicatorCommunity` 和 `IndicatorIDE` 的发布按钮均无处理器 | 用户无法发布指标到社区 |
| **策略市场跳转** | Dashboard 引导中的 "策略市场" 按钮无响应 | 新手引导流程断裂 |
| **合约平仓操作** | 合约页面有 "平仓" Tab 但无实际平仓逻辑 | 用户无法平掉合约仓位 |
| **资产划转/买币/兑换** | 合约交易页面 3 个按钮无响应 | 资金操作入口失效 |
| **AI 历史记录** | 始终显示空状态 | 用户无法查看 AI 分析历史 |
| **自选股搜索添加** | 搜索框无实际搜索逻辑 | 无法扩展关注列表 |

### 🟡 功能不完善

| 功能 | 现状 | 建议 |
|------|------|------|
| **风控规则加载** | `RiskControl.tsx` 中 `activeProtections` 初始为空数组，**未从服务器加载已有配置** | 页面加载时应调用 API 获取当前配置 |
| **i18n 国际化** | 整个项目 99% 文本为硬编码中文，`TopBar.tsx` 虽有语言切换但无实际翻译文件 | 接入 `react-i18next` 或至少提取文本常量 |
| **移动端导航** | `BottomNav` 仅 5 个入口（Dashboard/Trading/AI/Risk/Settings），大量页面在移动端无法访问 | 增加抽屉菜单或折叠入口 |
| **路由不一致** | Dashboard 中使用 `window.location.hash = '#/strategies'`，但 `App.tsx` 中路由为 `/strategy` | 统一路由命名 |
| **错误处理不统一** | 部分页面用 `alert()`（ModelManagement、IndicatorDetail），部分用 toast | 统一使用 `toast` |
| **类型安全** | 大量使用 `as any`，API 返回类型未定义 | 补充 TypeScript 接口定义 |

## 五、优化建议（按优先级排序）

### P0 — 立即修复（影响核心功能）

1. **补齐所有无响应按钮的 `onClick` 处理器**
   - Dashboard 引导按钮 → 跳转对应路由
   - IndicatorIDE 发布/另存为 → 对接 API
   - IndicatorCommunity 发布指标 → 打开发布表单/跳转 IDE
   - 合约交易 划转/买币/兑换 → 跳转或打开对应弹窗
   - AI 搜索 → 实现股票搜索并添加到 watchlist

2. **修复 Portfolio 盈亏日历导航方向**
   - 左箭头 → 上一月 (`i - 1`)
   - 右箭头 → 下一月 (`i + 1`)

3. **提取 CRA 参数表单为共享组件**
   - 新建 `src/components/strategy/CRAParamForm.tsx`
   - 5 个页面统一引用

### P1 — 近期优化（提升代码质量）

4. **提取交易页面共享逻辑**
   - `src/hooks/useTradingChart.ts` — KLineChartPro 初始化、周期监听
   - `src/components/trading/OrderBookPanel.tsx` — 订单簿渲染
   - `src/components/trading/TradeHistoryPanel.tsx` — 成交/委托/持仓表格

5. **统一 ECharts 加载**
   - 新建 `src/lib/echarts.ts` 导出 `getEcharts()`

6. **风控页面加载已有配置**
   - 页面挂载时调用 `protectionApi.config()` 获取当前规则

7. **统一错误处理**
   - 全局替换 `alert()` 为 `toast()`
   - `Billing.tsx` 改用 `axiosInstance`

### P2 — 中期规划（提升用户体验）

8. **移动端导航完善**
   - BottomNav 增加 "更多" 抽屉，包含 Bots、Backtest、Portfolio、ML 等入口
   - 或根据用户权限动态显示常用入口

9. **i18n 框架搭建**
   - 引入 `react-i18next`
   - 先提取所有中文文本到 `src/locales/zh-CN.json`

10. **类型安全加固**
    - 为 `api.ts` 中所有 API 命名空间补充返回类型接口
    - 逐步减少 `as any` 使用

11. **合约平仓功能实现**
    - 在 "平仓" 模式下显示当前持仓并提供平仓/减仓按钮
    - 调用 `orderApi.place()` 提交反向订单
